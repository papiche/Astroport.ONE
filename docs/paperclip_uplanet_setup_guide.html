Étape 04 · CLI & Interface
Skills & variables d'environnement
Les skills sont des scripts Node.js 18+ que les agents appellent pour interagir avec les canaux externes.
CLI
CLI Générer les skills via le script de configuration
paperclip-configure-channels.sh dépose les .js dans le volume monté
# Vérifier Node.js dans le conteneur (doit être v18+) docker exec ai-company-swarm-paperclip-1 node --version # Générer tous les skills et configurer les tokens ./paperclip-configure-channels.sh --all # Ou canal par canal : ./paperclip-configure-channels.sh --telegram ./paperclip-configure-channels.sh --gmail ./paperclip-configure-channels.sh --mastodon
Volume monté : les fichiers générés dans ~/.zen/ai-company/paperclip_data/skills/ sont directement visibles par le conteneur Paperclip sans redémarrage.
CLI
CLI Vérifier les skills générés
Trois fichiers .js — fetch natif Node.js, zéro dépendance npm
ls -la ~/.zen/ai-company/paperclip_data/skills/ # telegram_poster.js → fetch vers api.telegram.org # gmail_sender.js → OAuth2 refresh + MIME encode en JS pur # mastodon_poster.js → fetch vers instance Mastodon

Étape 08 · Interface
Inbox & Validation
L'Inbox est le mécanisme de gouvernance humaine — chaque action critique passe par toi avant d'être exécutée.
MODE
INTERFACE Configurer le mode d'approbation
Settings → Governance — garder MANUAL la première semaine
Settings → Governance → Approval gate MANUAL : tu valides chaque action avant exécution ← semaine 1 AUTO : les agents agissent sans attendre ton accord ← semaine 2+
Stratégie recommandée :
Semaine 1 : MANUAL — observer les décisions du CEO, corriger les personas si besoin.
Semaine 2 : passer en AUTO pour les actions de routine (posts, réponses Telegram).
Garder MANUAL pour les actions à fort impact (email blast, publication externe).
UI
INTERFACE Types de demandes dans l'Inbox
Ce que le CEO peut soumettre — et comment réagir
Hire Agent → vérifier le rôle et le budget proposés → Approve / Reject Use Tool → vérifier l'action (ex: envoyer un email) → Approve / Edit Create Ticket → vérifier le prompt et le heartbeat → Approve / Modify Spend Budget → vérifier si le montant est justifié → Approve / Reduce
Astuce : cliquer sur une demande pour voir l'intégralité du raisonnement du CEO — le "trace" montre exactement pourquoi il a pris cette décision.
UI
INTERFACE Lire l'audit log (Trace)
Chaque action est loggée et immuable — traçabilité complète
Tickets → cliquer sur un ticket exécuté → Tab "Trace" ou "History" → Voir dans l'ordre : 1. Prompt envoyé à l'agent 2. Tool calls effectués (skill appelé, paramètres) 3. Réponse du modèle 4. Output final sauvegardé

CLI
CLI Rapport de coût détaillé sur 30 jours
Requête directe PostgreSQL dans le conteneur
docker exec ai-company-swarm-postgres-1 \ psql -U paperclip -d paperclip -c " SELECT a.name AS agent, ROUND(SUM(t.token_cost)::numeric, 4) AS cout_usd, COUNT(t.id) AS nb_taches, ROUND(AVG(t.token_cost)::numeric, 5) AS cout_moyen FROM tasks t JOIN agents a ON t.agent_id = a.id WHERE t.created_at > NOW() - INTERVAL '30 days' GROUP BY a.name ORDER BY cout_usd DESC;"

Étape 10 · Interface & CLI
Concurrence & Scale
Augmenter la cadence une fois les agents calibrés — plusieurs agents en parallèle, modèles plus puissants si besoin.
UI
INTERFACE Augmenter la concurrence
Advanced Settings → Concurrency — permet plusieurs agents en parallèle
Company Settings → Advanced → Concurrency 1 → un agent à la fois (défaut, semaine 1) 2 → deux agents simultanés (semaine 2+) 4 → quatre agents (phase production)
Quand augmenter ? Quand les tickets s'accumulent en attente et que les agents sont calibrés (personas OK, budget validé, Inbox en AUTO). Augmenter progressivement : 1 → 2 → 4.
UI
INTERFACE Passer en automation complète
Semaine 3+ — une fois les personas stables et l'Inbox validée en AUTO
Semaine 1
Mode observation
Inbox MANUAL · Concurrency 1 · Valider chaque action CEO · Affiner les personas
Semaine 2
Mode semi-auto
Inbox AUTO pour routines · MANUAL pour actions email · Concurrency 2
Semaine 3+
Mode production
Inbox AUTO complet · Concurrency 4 · Vérification du rapport CEO quotidien seulement
