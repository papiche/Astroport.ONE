## 🎮 El Recorrido Iniciático: Del Curioso al Capitán

Astroport.ONE no es solo software — es un **camino de iniciación gamificado** del consumidor digital al copropietario cooperativo.

```
Día 0: Descubres UPlanet ORIGIN (entorno de pruebas)
       ↓ Instalación en un comando — cero configuración
Días 1–7: Recibes ZINEs diarios (tutoriales digitales)
       ↓ Aprende yt-dlp, IA #BRO, Nostr, economía Ẑen
Día 7: La Llamada del DRAGÓN — únete a la red certificada
       ↓ Valida tu formación → recibe tu swarm.key
       ↓ UPlanet ORIGIN  →  UPlanet ẐEN
```

El sistema **te envía automáticamente correos de coaching diarios** — cada ZINE adaptado a tu rol (Explorador, suscriptor ZENCard, Capitán, miembro U.SOCIETY).

## 🤖 #BRO: La Inteligencia del Enjambre

**#BRO** (Brain Robot Online) es la **IA colectiva** de la constelación Astroport:

- 🧠 **Local y privado**: funciona en cada estación mediante Ollama (sin datos enviados a OpenAI)
- 🎨 **Generación de imágenes** para tus publicaciones de Nostr
- 📝 **Redacción de artículos** en tu blog descentralizado
- 🔍 **Búsqueda semántica** en tu constelación
- 🌙 **Trabajo nocturno**: #BRO recopila y analiza mientras duermes

A diferencia de ChatGPT, #BRO es un **enjambre de inteligencias** — cada estación contribuye, el colectivo crece. Cuantos más Capitanes, más poderosa es la IA.

```bash
# Desde Nostr, cualquier mensaje etiquetado:
#BRO resume mis mensajes de la constelación
#BRO crea una imagen de: [descripción]
#BRO informe semanal

# API local (Ollama):
curl http://127.0.0.1:11434/api/generate -d '{"model":"llama3","prompt":"Hola BRO"}'
```

## 🛠️ Instalación

### Opción A — Docker Webtop (recomendado para probar sin modificar tu sistema)

```bash
cd docker/
docker compose -f docker-compose.webtop.yml up -d
# → http://localhost:3000  (escritorio en el navegador, Ubuntu XFCE + KasmVNC)
```

El contenedor webtop ejecuta `install.sh` automáticamente en el primer arranque (10–30 min). Todos los perfiles funcionan (`nextcloud`, `bleeding-edge`) porque el socket Docker se comparte con el host (arquitectura "sibling containers").

### Opción B — Instalación directa (bare-metal Linux — Debian/Ubuntu/Mint)

```bash
bash <(curl -sL https://install.astroport.com)
```

### Perfiles de instalación

```bash
# Estándar — IPFS + Nostr + UPassport + Astroport
bash install.sh

# + NextCloud AIO (nube privada 128 Go, lista para ZENCard)
bash install.sh "" "" "" nextcloud

# + Pila IA Enjambre (Ollama + Paperclip + OpenClaw + Qdrant)
bash install.sh "" "" "" bleeding-edge

# + rnostr dev (relay Rust, reemplaza strfry — experimental)
bash install.sh "" "" "" dev
```

| Perfil | Descripción | Estado |
|---|---|---|
| `standard` | IPFS · Nostr strfry · UPassport · GPG · gcli | ✅ Producción |
| `nextcloud` | Estándar + NextCloud AIO (128 Go, almacenamiento BTRFS) | ✅ Estable |
| `bleeding-edge` | Estándar + Paperclip + OpenClaw + LiteLLM + Qdrant IA | ⚠️ Experimental |
| `dev` | Estándar + relay Rust rnostr (migración de plugins bash writePolicy) | 🔧 En desarrollo |

## 💾 Arquitectura de Almacenamiento: BTRFS Recomendado

Para estaciones en producción, monta un **disco dedicado BTRFS** en `/nextcloud-data`:

```bash
# Formatear y montar
sudo mkfs.btrfs -L astrodata /dev/sdX
sudo mount -o compress=zstd,noatime /dev/sdX /nextcloud-data

# /etc/fstab (permanente)
UUID=$(blkid -s UUID -o value /dev/sdX)
echo "UUID=$UUID /nextcloud-data btrfs compress=zstd,noatime 0 0" | sudo tee -a /etc/fstab

# Migrar los datos
~/.zen/Astroport.ONE/stop.sh
sudo mv ~/.zen /nextcloud-data/zen   && ln -s /nextcloud-data/zen ~/.zen
sudo mv ~/.ipfs /nextcloud-data/ipfs && ln -s /nextcloud-data/ipfs ~/.ipfs
~/.zen/Astroport.ONE/start.sh
```

¿Por qué BTRFS? **Copy-on-Write** = IPFS nunca reescribe bloques → coincidencia perfecta. **Instantáneas** para copias de seguridad inmediatas de NextCloud. **Compresión zstd** ~25% de ahorro de espacio.

```
/nextcloud-data/
  zen/          ← enlace simbólico ~/.zen (datos Astroport, TW, game, players)
  ipfs/         ← enlace simbólico ~/.ipfs (bloques IPFS, 50-500 Go)
  nextcloud/    ← volúmenes NextCloud AIO (datos de usuario 128 Go cada uno)
```

## 💰 La Economía Cooperativa: El Modelo 3×1/3

Cada Ẑen recolectado por la estación se distribuye **automáticamente y de forma transparente** mediante el código:

```
Ẑen recolectados   →  1/3 Tesorería (reserva cooperativa)
(sin decisión           1/3 I+D (desarrollo de código abierto)
humana)                 1/3 Activos Comunes (tierras, bosques, infraestructura)
```

**Pagos semanales** (automatizados por `PLAYER.refresh.sh` y `NOSTRCARD.refresh.sh`):

| Fuente | Importe | Destinatario |
|---|---|---|
| Alquiler MULTIPASS | 1 Ẑen/semana | Estación del Capitán |
| Alquiler ZENCard | 4 Ẑen/semana | Estación del Capitán |
| PAF Armador | 14 Ẑen/semana | Anfitrión (desde la cooperativa) |
| PAF Capitán | 28 Ẑen/semana | Capitán certificado |
