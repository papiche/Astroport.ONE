Captain Jour 3 : Votre infrastructure

Votre station tourne sur 3 conteneurs Docker. Voici comment verifier, maintenir et activer chaque service.

Les 3 conteneurs Docker

  • astroport : IPFS + strfry + UPassport + APIs (toujours actif)
  • npm : Nginx Proxy Manager — SSL et reverse proxy (toujours actif)
  • nextcloud : NextCloud AIO 128 Go (optionnel, profil full)

Commandes de base :

docker compose ps
docker compose logs -f astroport
docker compose logs -f npm

Nginx Proxy Manager (NPM)

NPM gere le SSL et les sous-domaines :

  • astroport.DOMAIN : Station Map (port 12345)
  • ipfs.DOMAIN : IPFS Gateway (port 8080)
  • relay.DOMAIN : NOSTR Relay (port 7777)
  • u.DOMAIN : UPassport API (port 54321)
  • cloud.DOMAIN : NextCloud (port 8001)

Admin NPM : http://127.0.0.1:81 (local uniquement)

Activer NextCloud AIO (128 Go)

NextCloud utilise son propre docker-compose. Ports : 8443 (admin setup) | 8001 (app Apache) | 8002 (dashboard)

# Démarrer NextCloud AIO
docker compose \
  -f ~/.zen/Astroport.ONE/_DOCKER/nextcloud/docker-compose.yml \
  up -d

# Setup initial (première fois) — entrez votre domaine
https://127.0.0.1:8443

# NPM proxy cloud.DOMAIN → :8001 (auto si relancé)
~/.zen/Astroport.ONE/install/setup/setup_npm.sh

# Vérifier
docker logs nextcloud-aio-mastercontainer -f
  • Setup AIO : → https://127.0.0.1:8443 → entrez cloud.VOTREDOMAINE
  • NPM crée automatiquement cloud.DOMAIN → :8001 (relancez setup_npm.sh si besoin)
  • Mot de passe NPM : ~/.zen/nginx-proxy-manager/data/.admin_pass

💾 Stockage BTRFS — Architecture recommandée

Montez un disque dédié en BTRFS sur /nextcloud-data pour héberger tous vos volumes :

# Formater en BTRFS (remplacer /dev/sdX)
sudo mkfs.btrfs -L astrodata /dev/sdX
sudo mount -o compress=zstd,noatime /dev/sdX /nextcloud-data

# /etc/fstab (persistance au redémarrage)
UUID=$(blkid -s UUID -o value /dev/sdX)
echo "UUID=$UUID /nextcloud-data btrfs \
  compress=zstd,noatime 0 0" | sudo tee -a /etc/fstab

BTRFS = CoW (IPFS efficient) + snapshots + compression zstd ~25%

SSL NPM : Let's Encrypt (public) | auto-signé (local) — automatique via setup_npm.sh

🔀 Migrer ~/.zen et ~/.ipfs vers BTRFS

Une fois le disque BTRFS monté sur /nextcloud-data :

# 1. Arrêter Astroport
~/.zen/Astroport.ONE/stop.sh

# 2. Déplacer les données volumineuses
sudo mv ~/.zen /nextcloud-data/zen
sudo mv ~/.ipfs /nextcloud-data/ipfs

# 3. Créer les liens symboliques
ln -s /nextcloud-data/zen ~/.zen
ln -s /nextcloud-data/ipfs ~/.ipfs

# 4. Redémarrer
~/.zen/Astroport.ONE/start.sh

Vérifier : df -h /nextcloud-data | btrfs fi df /nextcloud-data
