Captain Day 4: Your constellation

Your station is not alone. The N2 Protocol automatically synchronizes NOSTR events between stations in your swarm.

The private IPFS swarm

All stations sharing the same swarm.key form a swarm:

  • Data replicated between stations
  • Encrypted P2P tunnels (DRAGON_p2p_ssh.sh)
  • Automatic peer discovery
# Check peers
ipfs swarm peers | wc -l
# View connections
ipfs swarm peers

Constellation backfill

The backfill_constellation.sh script synchronizes NOSTR events between relays:

  • Every hour: sync of filtered kinds (0, 1, 3, 7...)
  • N2: friends of friends of your users
  • Geo Keys: UMAP/SECTOR/REGION journals

NIP-101 script: backfill_constellation.sh in the NIP-101/ repo

Constellation monitoring

Each station publishes a kind 30850 event with its economic status:

  • economy.Swarm.html: real-time constellation view
  • Balance sheet, revenue, number of users per station
  • Health: healthy / warning / critical / bankrupt
  • Shared cooperative wallets (CASH, R&D, ASSETS)

Cooperative wallets are shared across all stations in the same swarm.

Adding a station

To expand your constellation:

  • 1. Find a new Shipowner (fiber + space)
  • 2. Install Astroport.ONE on the machine
  • 3. Copy the same swarm.key
  • 4. The new captain creates their MULTIPASS
  • 5. Stations discover each other automatically

Installation: git clone; ./install.sh; ./start.sh

Invoicing and reimbursement
