Captain Jour 4 : Votre constellation

Votre station n'est pas seule. Le Protocole N2 synchronise automatiquement les evenements NOSTR entre les stations de votre essaim.

Le swarm IPFS prive

Toutes les stations qui partagent le meme swarm.key forment un essaim :

  • Donnees repliquees entre stations
  • Tunnels P2P chiffres (DRAGON_p2p_ssh.sh)
  • Decouverte automatique des pairs
# Verifier les pairs
ipfs swarm peers | wc -l
# Voir les connexions
ipfs swarm peers

Backfill constellation

Le script backfill_constellation.sh synchronise les evenements NOSTR entre relays :

  • Toutes les heures : sync des kinds filtres (0, 1, 3, 7...)
  • N2 : amis des amis de vos utilisateurs
  • Geo Keys : journaux UMAP/SECTOR/REGION

Script NIP-101 : backfill_constellation.sh dans le repo NIP-101/

Monitoring constellation

Chaque station publie un event kind 30850 avec son etat economique :

  • economy.Swarm.html : vue constellation temps reel
  • Bilan, revenus, nombre d'utilisateurs par station
  • Sante : healthy / warning / critical / bankrupt
  • Portefeuilles cooperatifs partages (CASH, R&D, ASSETS)

Les portefeuilles cooperatifs sont partages entre toutes les stations du meme essaim.

Ajouter une station

Pour agrandir votre constellation :

  • 1. Trouvez un nouvel Armateur (fibre + espace)
  • 2. Installez Astroport.ONE sur la machine
  • 3. Copiez le meme swarm.key
  • 4. Le nouveau capitaine cree son MULTIPASS
  • 5. Les stations se decouvrent automatiquement

Installation : git clone; ./install.sh; ./start.sh

Facturation et remboursement

Convertissez vos Zen en euros via le G1FabLab :

  • 1. Le protocole mesure votre activite (hebergement, maintenance)
  • 2. Emettez une facture sur opencollective.com/monnaie-libre
  • 3. Le collectif valide et vire par SEPA
  • 4. Declarez en BNC (auto-entrepreneur)

SIRET requis. TVA non applicable si micro-entreprise.
