Prerequis techniques

Administrer une station Astroport.ONE demande des competences en :

  • BASH : scripts, variables, pipes, cron, sed/awk
  • Docker : conteneurs, compose, volumes, reseaux
  • Python venv : environnements virtuels (~/.astro/)
  • Linux : systemd, journalctl, SSH, permissions
  • Reseau : ports, DNS, certificats SSL, IPFS

Evaluez-vous avant de continuer.

Formations recommandees

Si vous n'etes pas a l'aise, commencez par :

  • BASH : pad.p2p.legal/BASH
  • SHELL : pad.p2p.legal/s/SHELL
  • Docker : pad.p2p.legal/Docker

Le code Astroport.ONE est 95% bash. Comprendre my.sh, les variables d'environnement et le cycle 20h12.process.sh est essentiel.

ORIGIN vs ZEN

  • ORIGIN : bac a sable ouvert. swarm.key = 0000. 1 Zen = 0.1 G1. Pas de swarm prive.
  • ZEN : production. swarm.key unique. 1 Zen = 1 EUR. Services cooperatifs complets.

Pour passer en mode ZEN :

  • Generez un swarm.key unique
  • Redistribuez-le a vos stations
  • Les comptes ORIGIN restent (migration automatique)
