🔐 Pour les Curieux : Comment Fonctionne l'Authentification SSSS ?

💡 En résumé : Votre carte contient 1 fragment sur 3 de votre clé privée. Il faut 2 fragments pour vous authentifier. Même si quelqu'un vole votre carte, il ne peut rien faire sans le 2ème fragment stocké sur le réseau.

Lorsque vous scannez votre QR code SSSS sur un terminal UPlanet, voici les opérations cryptographiques qui se déroulent en arrière-plan :

📱 Étape 1 : Détection et Transmission

Le terminal détecte que votre QR code commence par M- ou 1- (format SSSS MULTIPASS) et extrait :

  • SSSS1 : La première part du secret Shamir (dans le QR code)
  • IPNSVAULT : La clé IPNS (k51qzi5uqu5d...) qui identifie votre coffre-fort décentralisé
  • PASS : Le code à 4 chiffres que vous entrez (1111, 9999, 7777, 0000, ou vide)

🔍 Étape 2 : Recherche du MULTIPASS

Le système recherche votre MULTIPASS dans la base locale en utilisant l'IPNSVAULT. Si votre MULTIPASS n'est pas trouvé sur ce terminal, vous verrez le message "NOSTR CARD MISSING".

🔓 Étape 3 : Décryptage SSSS (Shamir Secret Sharing Scheme)

Le système combine deux parts du secret pour reconstruire votre clé :

  1. Part 1 (SSSS1) : Extraite directement de votre QR code scanné
  2. Part 2 (TAIL) : Décryptée depuis ssss.tail.uplanet.enc en utilisant la clé UPlanet

Ces deux parts sont combinées via ssss-combine pour reconstruire le secret DISCO qui contient votre salt et pepper.

🔐 Sécurité : Le système utilise un schéma "2 sur 3" : il faut au minimum 2 parts sur 3 possibles (HEAD=utilisateur, MIDDLE=UPlanet, TAIL=Capitaine) pour reconstruire le secret. Votre carte ne contient qu'une seule part, insuffisante seule pour compromettre votre identité.

🔑 Étape 4 : Génération des Clés

À partir du salt et pepper reconstruits, le système génère :

  • NSEC : Votre clé privée NOSTR (pour l'authentification réseau social décentralisé)
  • DUNIKEY : Votre clé privée Ğ1 (pour les transactions monétaires, si nécessaire)

💡 Important : Ces clés sont générées à la volée et ne sont jamais stockées dans le navigateur. Elles restent uniquement en mémoire pendant la session.

🎯 Étape 5 : Routage selon le Code PASS

Selon le code PASS que vous avez entré, le système ouvre différentes interfaces :

  • PASS vide → Interface simple de message NOSTR (nostr.html)
  • PASS 1111 → Interface demo Astro Base avec messenger NOSTR (astro_base.html)
  • PASS 9999 → Terminal de paiement MULTIPASS avec solde ẐEN (scan_multipass_payment.html)
  • PASS 7777 → Enregistreur vidéo webcam avec authentification NOSTR (webcam.html)
  • PASS 0000 → Résiliation : vide le portefeuille Ğ1 et régénère le MULTIPASS
🛡️ Avantages de cette Architecture :
  • Aucun stockage navigateur : Vos clés ne sont jamais sauvegardées dans le navigateur
  • Authentification sans mot de passe : La possession physique de votre carte suffit
  • Multi-terminal : Votre MULTIPASS fonctionne sur n'importe quel terminal UPlanet
  • Résilience : Même si une part est compromise, il faut encore une deuxième part pour reconstruire le secret
  • Régénération possible : En cas de perte, utilisez le code 0000 pour récupérer vos fonds et créer un nouveau MULTIPASS
